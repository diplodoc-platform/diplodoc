Улучшаем документацию

TechWriter Days 3 · Задание от Diplodoc

В чем суть

В рамках практикума вам предложен вымышленный кейс. Некоторый стартап планирует выпустить новый облачный API для обработки изображений. Сервис называется PixelPush — он умеет сжимать картинки, менять размер, накладывать водяные знаки и обрабатывать файлы пачками. Пользователи будут подключать его, чтобы не поднимать собственную инфраструктуру.

Бэкенд-разработчик проявил инициативу и набросал первую версию документации. Ваша задача — превратить черновик в удобный документационный проект, используя возможности Diplodoc. Какие именно фичи использовать — решайте сами.

Работы нужно прислать сегодня, до 16 часов.

Целевая аудитория будущей документации

Документацией по PixelPush будут пользоваться:

  • Разработчики — интегрируют API в свои сервисы. Знают HTTP, JSON, умеют читать код. Им важны точные параметры, примеры на их языке и быстрый доступ к справочнику.
  • Менеджеры продукта и маркетологи — оценивают возможности сервиса, готовят материалы для клиентов. Не пишут код, но должны понять что умеет PixelPush, какие есть тарифы и ограничения.

Как будет оцениваться результат

Мы будем смотреть на то, насколько удобно получившийся документ читать: логична ли структура, хорошо ли выделено важное, уместно ли использованы фичи. Чем больше возможностей Diplodoc задействовано к месту — тем лучше.

Подходите к задаче творчески — здесь нет единственно правильного ответа. И пожалуйста, без ИИ-помощников: интересно посмотреть на ваше собственное решение.

🎁 Призы Лучшие работы не останутся без подарков.

Как начать

Если вы работаете на своём ноутбуке
  1. Склонируйте к себе репозиторий с проектом: 
    git clone https://github.com/ogorbacheva/diplodoc.git ~/diplodoc
  2. Перейдите в папку make-docs-better: 
    cd ~/diplodoc/make-docs-better
    В папке docs/ находится документация, которую вам и предстоит улучшить:
    docs/
    ├── .yfm      # конфиг проекта
    ├── toc.yaml  # навигация
    └── index.md  # драфт от разработчика
  3. Соберите документацию: 
    yfm -i docs -o output
    Зайдите в папку output и откройте в браузере страничку index.html — это собранная документация.
  4. Теперь можно начать улучшать документацию.
Если вы работаете на выданном ноутбуке
  1. Перейдите в папку проекта make-docs-better: 
    cd ~/Desktop/workshop/diplodoc/make-docs-better
    В папке docs/ находится документация, которую вам и предстоит улучшить:
    docs/
    ├── .yfm      # конфиг проекта
    ├── toc.yaml  # навигация
    └── index.md  # драфт от разработчика
  2. Соберите документацию: 
    yfm -i docs -o output
  3. Теперь можно начать ее улучшать!

Как отправить результат

Если вы работаете на своём ноутбуке

Заархивируйте папку make-docs-better и пришлите архив на почту: mkoshkin@yandex-team.ru

Если вы работаете на выданном ноутбуке
  1. Сохраните ваши изменения в файле (cmd+S).
  2. В терминале запустите скрипт: 
    cd ~/Desktop/workshop/diplodoc/
./save <ваш_логин_в_тг>
    Этот скрипт скопирует вашу работу в отдельную папку и приведёт текущее состояние проекта к изначальному виду.
