Улучшаем документацию

TechWriter Days 3 · Задание от Diplodoc

Суть игры

Добро пожаловать на практикум! Представьте себя в роли технического писателя, который только что устроился в развивающийся стартап. Ваша первая рабочая задача — навести порядок в документации, которую разработчик написал «на скорую руку».

На время этой игры вам будет предложен вымышленный сценарий. Он придуман специально для конференции, однако проблема, с которой вы столкнётесь, знакома каждому из нас: как превратить простыню текста в структурированный и удобный продукт.

Что нужно сделать

Улучшите проект с помощью фич Diplodoc: выделите важное, структурируйте контент, добавьте навигацию. Какие именно фичи использовать и где — решайте сами.

Описание кейса

Некоторый стартап планирует выпустить новый облачный API для обработки изображений. Сервис называется PixelPush — он умеет сжимать картинки, менять размер, накладывать водяные знаки и обрабатывать файлы пачками. Пользователи будут подключать его, чтобы не поднимать собственную инфраструктуру.

Бэкенд-разработчик написал документацию к этому API — подробно и по делу. Весь контент есть, но оформлен как черновик: один большой файл без навигации, без структуры. Ваша задача — превратить его в удобный документационный проект, используя возможности Diplodoc.

Целевая аудитория будущей документации

Документацией по PixelPush будут пользоваться:

  • Разработчики — интегрируют API в свои сервисы. Знают HTTP, JSON, умеют читать код. Им важны точные параметры, примеры на их языке и быстрый доступ к справочнику.
  • Менеджеры продукта и маркетологи — оценивают возможности сервиса, готовят материалы для клиентов. Не пишут код, но должны понять что умеет PixelPush, какие есть тарифы и ограничения.

Как начать

1. Получите исходный проект:

  • Если вы работаете на выданном ноутбуке — папка make-docs-better уже лежит на Рабочем столе.
  • Если у вас свой ноутбук — склонируйте репозиторий:
    git clone --branch techwriter-days-26 --single-branch https://github.com/ogorbacheva/diplodoc.git
cd diplodoc/make-docs-better

2. Создайте папку с вашим Telegram-логином на рабочем столе — например, ivan_petrov — и скопируйте туда make-docs-better.

3. Откройте терминал, перейдите в свою папку и соберите проект:

cd ~/Desktop/ivan_petrov/make-docs-better
yfm -i docs -o output

Папка output создастся автоматически. Откройте output/index.html в браузере — это ваша отправная точка.

Исходный проект в папке docs/:

docs/
├── .yfm      # конфиг проекта
├── toc.yaml  # навигация
└── index.md  # вся документация одним файлом

Документация Diplodoc

Как оценивается результат

Мы будем смотреть на то, насколько удобно получившийся документ читать: логична ли структура, хорошо ли выделено важное, уместно ли использованы фичи. Чем больше возможностей Diplodoc задействовано к месту — тем лучше.

Подходите к задаче творчески — здесь нет единственно правильного ответа. И пожалуйста, без ИИ-помощников: интересно посмотреть на ваше собственное решение.
